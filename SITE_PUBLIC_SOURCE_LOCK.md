## Working rule

These public sources are the reference before any future modification.

No page should be treated as empty just because the GitHub `main` file appears empty.
No publication or merge should replace existing public sections, images, tools, guides, buttons, menus, footer, or responsive behavior.

Future work must be done by:
1. checking the public source backup,
2. identifying the exact section to modify,
3. applying a micro-change only,
4. preserving all existing content and visual assets,
5. comparing before/after before merge.

The branch `safe-integration-pages` is a draft and must not be merged as-is.
